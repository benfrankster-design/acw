# Goals — ACW

ACW's goals are documented in depth elsewhere; this file is the lightweight pointer.

- **Long-arc:** ship ACW as a teaching template + methodology archive that helps any operator stand up persistent agentic workspaces with governance from session zero. See `research/01-problem-framing.md` for the original framing.
- **Near-term (current quarter-equivalent):** earn-by-incident loop running cleanly across the lattice. Each downstream instance dogfood produces evidence; v0.N → v0.N+1 absorbs what generalizes.
- **Endpoint vision:** an organization can run from a federation of coordinated ACW instances (org-brain + departmental). See `rules/multi-instance-topology.md`.

## Out of scope

- Production-grade multi-tenant infrastructure. ACW is a teaching template; the deferred primitives are design documents, not validated implementations.
- Vendor-specific tooling. ACW is plain-text, stdlib-only, model-agnostic.
- Replacing existing controlled-vocabulary or governance tools. ACW is the minimum-viable layer; mature tools (SKOS, Vault, etc.) port in when their infrastructure is available.
